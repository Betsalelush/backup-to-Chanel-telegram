# 📦 Backup to Channel Telegram

אוסף סקריפטים בפייתון לגיבוי והעברת הודעות בין ערוצי וקבוצות טלגרם, מבוסס על ספריית **Telethon**.

הסקריפטים מאפשרים גיבוי ערוצים שלמים (כולל מדיה), העברה בין ערוצים פרטיים וציבוריים, עבודה עם מספר חשבונות במקביל, חיבור דרך Tor, ועוד.

---

## 📋 דרישות מקדימות

לפני שמריצים את הסקריפטים, צריך להגדיר:

1. **API ID ו-API Hash** מ־[my.telegram.org](https://my.telegram.org)
   → להזין אותם בקוד או להגדיר כמשתני סביבה (`API_ID`, `API_HASH`).
2. **Python 3.8+** עם הספריות:
   ```bash
   pip install telethon pyshorteners qrcode rich PySocks
   ```
3. עבור `tor.py` בלבד — להפעיל **Tor** (פורט 9050) לפני הריצה.

---

## 📂 מבנה הקבצים

| קובץ | תיאור קצר |
|------|-----------|
| `seshenqr.py` | יצירת קובץ סשן (חיבור לטלגרם) באמצעות **קוד QR** |
| `bob.py` | מעביר הודעות מערוץ לערוץ — גרסת בסיס |
| `boba.py` | העברה מערוצים ציבוריים + בחירת ID התחלה |
| `boby.py` | מיועד לערוצים עם **הגבלת העברה** — מוריד ומעלה כל הודעה מחדש |
| `meudcan.py` | מעביר הודעות עם המשך מההתקדמות האחרונה / מההתחלה |
| `meudcan2.py` | כמו `meudcan.py`, עם אפשרות נוספת להתחיל ממספר הודעה ספציפי |
| `tor.py` | גרסה מתקדמת — מספר חשבונות במקביל + Tor + סינון סוגי קבצים |
| `lo.py` | שולח קישור מקוצר (TinyURL/Bitly) לבוט — לעקיפת חסימת קישורים בטלגרם |

## 🔧 פירוט הקבצים

### 🔑 `seshenqr.py` — יצירת סשן עם QR
מייצר מחרוזת סשן (`session_string`) ע"י סריקת קוד QR מאפליקציית הטלגרם:
- מבקש API ID, API Hash ומספר טלפון.
- מציג קוד QR בטרמינל לסריקה (`Settings → Devices → Scan QR code`).
- תומך באימות דו-שלבי (2FA).
- שומר את הסשנים בקובץ `sessions.json` — בו משתמש `tor.py`.
- ניתן ליצור מספר סשנים ברצף (תמיכה במספר חשבונות).

---

### 📨 `bob.py` — מעביר בסיסי בין ערוצים
מעביר הודעות בין שני ערוצים שהמשתמש חבר בשניהם:
- מעתיק טקסט ומדיה (עד 2GB) **ללא קרדיט** למקור (באמצעות `send_message` עם אובייקט ההודעה).
- שמירת התקדמות בקובץ `התקדמות.json`.
- הגבלת קצב מובנית: 20 הודעות לדקה + השהיה של 2 שניות בין הודעות.
- טיפול ב-`FloodWaitError` עם המתנה אוטומטית.

---

### 📨 `boba.py` — העברה מערוצים ציבוריים
דומה ל-`bob.py`, עם תמיכה משופרת בערוצים ציבוריים:
- מקבל קישור (`https://t.me/c/.../...`), שם משתמש (`@channel`) או ID מספרי.
- עבור **ערוץ ציבורי עם שם משתמש** (`@channel`) — לא חייבים להיות חברים בו.
- עבור **ערוץ פרטי** או קישור `t.me/c/...` — חייב להיות חבר בערוץ.
- לאחר תחילת התהליך, אפשר לצאת מהערוץ והגיבוי ימשיך לרוץ (Telethon שומר את ה-entity ב-cache).
- אפשרות לבחור את **ID ההודעה** ממנו להתחיל — שימושי לדילוג על הודעות שכבר הועברו.

> 📝 בעת בחירת ID התחלה, מונה ההודעות מתאפס לאפס (לתצוגה בלבד) — אבל הגיבוי עצמו ממשיך מהמיקום שנבחר.

---

### 📥 `boby.py` — העברה לערוצים עם הגבלת Forwarding
מיועד לערוצים שבהם **הגבלת העברה** (Restrict saving content) פעילה:
- במקום להעביר ישירות, מוריד כל הודעה (טקסט + מדיה) ומעלה אותה כחדשה לערוץ היעד.
- שומר את כל הטקסט ככיתוב (caption) של הקובץ.
- מנקה את הקבצים הזמניים אחרי השליחה.
- זהה ל-`bob.py` במנגנוני הגבלת קצב, התקדמות וטיפול בשגיאות.

---

### 📨 `meudcan.py` — בסיס פשוט
גרסה מינימליסטית של מעביר ההודעות:
- שתי אפשרויות בלבד: המשך מהאחרונה / התחלה מאפס.
- ללא בחירת ID התחלה ספציפי.

---

### 📨 `meudcan2.py` — גרסה משופרת של `meudcan.py`
זהה ל-`meudcan.py`, עם תוספת:
- אפשרות 3: **התחל ממספר הודעה ספציפי** — מקלידים ID של הודעה והגיבוי מתחיל ממנה.

---

### 🌐 `tor.py` — הגרסה המתקדמת ביותר
סקריפט מקיף בעל יכולות מתקדמות:
- **ריבוי חשבונות**: טוען את כל הסשנים מ-`sessions.json` (שנוצרו ע"י `seshenqr.py`) ועובד איתם במקביל.
- **חיבור דרך Tor** (אופציונלי, פר חשבון, דרך SOCKS5 על פורט 9050).
- **סינון סוגי קבצים**: בחירה האם להעביר טקסט בלבד / תמונות / וידאו / אודיו / מסמכים / הכל / מותאם אישית.
- **חלוקת עומס** הוגנת בין החשבונות (round-robin עם ניהול FloodWait פר חשבון).
- **השהיה דינמית**: מתקצרת אחרי הצלחות רצופות, מתארכת לאחר כישלונות.
- **בדיקת שליחה מוקדמת** לכל חשבון לערוץ היעד לפני התחלת הגיבוי.
- תמיכה ב**ערוצי פורום** (שולח לנושא הכללי).
- שמירת התקדמות מתקדמת ב-`progress.json` (כולל set של ID-ים שכבר נשלחו).
- חיפוש אוטומטי בדיאלוגים אם הזיהוי הישיר של הערוץ נכשל.
- הצגת רשימת ערוצים זמינים לבחירה.

> 🚀 זה הסקריפט המומלץ לפרויקטים גדולים או רגישים שדורשים אנונימיות וביצועים.

---

### 🔗 `lo.py` — שליחת קישור מקוצר לבוט
שירות נלווה לעקיפת חסימת קישורים בטלגרם:
- מקצר קישור ארוך באמצעות **TinyURL** (עם נפילה אוטומטית ל-**Bitly** במידת הצורך).
- שולח לבוט (לפי בחירה) פקודה + הקישור המקוצר (לדוגמה: `/leech2 https://tinyurl.com/...`).
- כולל ניסיונות חוזרים (עד 3) וטיפול ב-`FloodWait`.
- שימושי במיוחד לבוטים מסוג `leech` שטלגרם חוסמת להם קישורים ישירים.

## 🐛 באגים ידועים

### בלבול בין ערוצים בשמירת ההתקדמות
**רלוונטי לכל סקריפטי הגיבוי** (`bob.py`, `boba.py`, `boby.py`, `meudcan.py`, `meudcan2.py`, `tor.py`).

קובץ ההתקדמות (`התקדמות.json` / `progress.json`) שומר רק `last_message_id` אחד גלובלי, בלי לשייך אותו לערוץ המקור הספציפי. כתוצאה מכך:
- אם מתחילים גיבוי של ערוץ A עד הודעה מספר 5000, עוצרים, ועוברים לגיבוי של ערוץ B —
- הסקריפט יקרא את ההתקדמות של A וידלג על כל הודעות B עד מספר 5000.

**עקיפה זמנית:** למחוק או לשנות את שם קובץ ההתקדמות לפני מעבר לערוץ אחר, או לבחור באפשרות "התחל מההתחלה".

**תיקון רצוי:** לשנות את מבנה ה-JSON כך שיהיה מילון לפי ID של ערוץ המקור, לדוגמה:
```json
{
  "channel_id_1": { "הודעה_אחרונה": 5000, "סך_הועברו": 5000 },
  "channel_id_2": { "הודעה_אחרונה": 1200, "סך_הועברו": 1200 }
}
```

## 🚀 שימוש מומלץ — Workflow

1. **יצירת סשן (פעם אחת):**
   ```bash
   python seshenqr.py
   ```
   סורקים את ה-QR ומקבלים `sessions.json`.

2. **בחירת הסקריפט המתאים:**
   - גיבוי פשוט בין שני ערוצים → `bob.py`
   - גיבוי ערוץ ציבורי עם שם משתמש → `boba.py`
   - גיבוי ערוץ עם הגבלת העברה → `boby.py`
   - גיבוי גדול במספר חשבונות + Tor → `tor.py`

3. **הרצה:**
   ```bash
   python <script_name>.py
   ```
   הסקריפט יבקש מספר טלפון (או ישתמש בסשן הקיים), אימות וקוד SMS אם צריך, ואז את ערוצי המקור והיעד.
